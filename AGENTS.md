# Tower Lite Agent Notes

- 前端 UI 按轻量工作台风格实现：白色表单底、浅灰边框、8px 左右圆角、蓝色 focus ring、克制间距。
- 新增 `input`、`select`、`textarea`、按钮时优先复用现有样式类；不要临时写一套和现有控件不一致的外观。
- 应用内可见下拉菜单必须使用项目内自定义下拉组件，不使用原生 `<select>`；避免出现系统黑色下拉层。
- 人员下拉选项显示头像、名字和邮箱；优先级下拉选项显示对应优先级颜色。
- 侧边栏宽度固定为 280px；侧边栏内所有表单、列表、下拉框和长文本都必须设置 `min-width: 0` / `max-width: 100%`，不能撑破边栏。
- 取消和危险操作按钮使用统一红色实心按钮、白色文字；确认操作使用统一蓝色实心按钮。
- 按钮元素里的文字全局禁止换行；按钮宽度不足时应调整布局、宽度或文案，不允许把按钮文字挤成两行。
- 如果用 `button` 承载任务卡、列表行等整块可点击内容，必须为内部标题、标签和容器设置宽度约束、截断或换行策略，不能让内容撑破父布局。
- 当前处于开发阶段，不需要为旧数据结构做兼容。数据库结构变化时按项目文档重置开发数据库。
